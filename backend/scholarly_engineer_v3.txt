1. Abstract
The rapid evolution of Electronics Engineering (ECE) demands a structured, accessible, and data-driven approach to academic learning. This project introduces “The Scholarly Engineer,” an interactive web application designed to curate, visualize, and analyze the ECE curriculum. Built entirely on a Python-based monolithic architecture using the Streamlit framework, the platform serves as a digital curator for electronics engineering students.
The application features an interactive hierarchical tree-based learning roadmap rendered via the streamlit-echarts library, a centralized repository for semester-wise study materials, a hub for practical hardware projects, and a dedicated data analytics dashboard. The roadmap module employs a custom Python algorithm to categorize each subject in the curriculum and dynamically assign color-coded ECharts nodes. Clicking any subject node triggers an inline analytics panel displaying a Matplotlib pie chart of that subject’s Lecture, Tutorial, and Practical hour distribution.
By integrating Python data science libraries — Pandas, NumPy, Matplotlib, and Seaborn — the platform provides students and educators with deep insights into workload distribution, credit allocation, and theoretical versus practical hours across all eight semesters.
This project bridges the gap between static academic syllabus documents and modern, interactive digital learning environments. It was developed as part of the Python Programming course for first-year second-semester students of the Electronics Engineering Department at Birla Vishvakarma Mahavidyalaya (BVM Engineering College), Vallabh Vidyanagar, for the academic year 2025–2026, under the guidance of Dipak M Patel.

2. Introduction
2.1 Background and Motivation
Engineering students often struggle to visualize the full progression of their degree. They lack a holistic view of how foundational subjects such as Semiconductor Physics and Network Analysis in the early semesters connect to advanced specializations like VLSI Design or Embedded Systems in the later years. Traditional curriculum delivery relies on static PDF files and fragmented resource sharing, making it difficult for students to plan their academic journey effectively.
Modern educational technology is moving away from static Learning Management Systems (LMS) toward interactive, student-centric dashboards. Platforms like Roadmap.sh have proven the effectiveness of visual learning paths for software engineers. However, hardware and electronics engineering disciplines lack a similarly rigorous, curated, and analytically rich visual guide. Furthermore, course workloads are rarely quantified in an accessible, subject-level manner.
2.2 Overview of the Proposed System
“The Scholarly Engineer” is a comprehensive, interactive digital platform designed specifically for students pursuing a B.Tech in Electronics Engineering. The application is divided into the following functional modules:
Interactive Curriculum Tree: A hierarchical tree visualization rendered using streamlit-echarts, displaying the full 8-semester ECE curriculum. Each subject node is color-coded by category (Core Subject, Elective, Project/Research, Lab/Practical, Mathematics) and clicking a node triggers a subject-level analytics panel.
Study Materials Repository: A semester-wise library linking directly to syllabi and reference materials hosted on cloud storage.
Project Hub: A curated collection of practical hardware projects categorized by difficulty, bridging the gap between theory and hands-on implementation.
Curriculum Analytics Dashboard: A data-driven module using Pandas, NumPy, Matplotlib, and Seaborn to visualize credit distributions and lecture-to-practical ratios across all eight semesters.
2.3 Significance of the Project
This project holds significant educational and practical value. It simultaneously demonstrates the power of Python as a full-stack development tool, integrating web framework capabilities (Streamlit) with data science libraries (Pandas, NumPy) and visualization tools (Matplotlib, Seaborn, streamlit-echarts) within a single unified application. The deliberate choice of a monolithic Streamlit architecture — as opposed to a distributed React/FastAPI setup — was made to maximize simplicity, maintainability, and educational clarity. The project directly applies Python programming concepts taught in the course to a real and meaningful problem in the academic domain.
2.4 Scope of the Project
The scope is defined by the B.Tech Electronics Engineering structure (2024–25), encompassing 8 semesters of coursework, open electives, program electives, and project-based learning modules. The platform is a scalable web application built entirely in Python that can be expanded to include other engineering disciplines in the future. The project covers Python web application development, hierarchical JSON data modeling, interactive chart rendering with streamlit-echarts, graph event handling between JavaScript and Python, data manipulation with Pandas, statistical aggregation with NumPy, and data visualization with Matplotlib and Seaborn.

The parchment-toned background (#fbf9f6) combined with deep indigo (#15236d) typography establishes a professional academic aesthetic. 4.2 Interactive Curriculum Tree
The interactive curriculum roadmap, rendered using streamlit-echarts, correctly displays the full B.Tech ECE curriculum as an expandable and collapsible horizontal tree. The root node (B.Tech Electronics Eng) branches into First Year through Fourth Year, each subdivided into individual semesters and their constituent subjects. The tree initializes at a depth of one level, allowing students to progressively expand branches of interest.
Two significant enhancements characterize this module:
Dynamic Color Coding via inject_colors()
A custom recursive Python function, inject_colors(), traverses the entire curriculum JSON tree before it is passed to the ECharts component. For each leaf node (i.e., each individual subject), the function inspects the node’s name and metadata to assign it to one of five categories: Core Subject (blue, #5470c6), Elective (green, #91cc75), Project/Research (yellow, #fac858), Lab/Practical (red, #ee6666), or Mathematics (light blue, #73c0de). Semester and year grouping nodes are rendered in dark grey with a white fill and a colored border to visually distinguish them from subject leaves. These colors are injected directly into the ECharts node’s itemStyle dictionary and are reflected in the rendered tree as filled circles at each node, with a corresponding color legend displayed at the top of the page.
Interactive Analytics Panel
A JavaScript click event handler is registered on the ECharts component via the streamlit-echarts events parameter. When a student clicks on any subject node, the node’s full data dictionary — including its name, metadata (Credits, L, T, P, H), and assigned category — is returned to the Streamlit Python backend as the selected_node variable. The right-hand analytics column then dynamically renders a color-coded banner identifying the subject and its category, displays its credit and hour summary, and generates a Matplotlib pie chart breaking down the distribution of Lecture (L), Tutorial (T), and Practical (P) hours for that specific subject. Subjects with zero hours in a particular category are correctly excluded from the pie chart.
4.3 Study Materials Repository
The semester selection widget dynamically renders course codes and subject names for each selected semester. For Semesters 1 and 2, download links to cloud-hosted syllabi function correctly. Semesters 3 through 8 display appropriately disabled UI states indicating upcoming content availability. The Pandas DataFrame underlying the repository allows rapid querying and renders correctly in all tested browsers.
4.4 Curriculum Analytics Dashboard
The data analytics module produced the following key results upon testing:
The Seaborn credit distribution bar chart correctly displayed varying credit weights across 8 semesters, with Semester 7 registering the highest credit load (24 credits) as expected for final-year specialization courses.
The Matplotlib dual-line plot accurately depicted the pedagogical shift from high lecture hours in Semesters 1–4 to high practical hours in Semesters 7–8, validating the data pipeline from Pandas through to visualization.
NumPy aggregation functions correctly computed total degree credits (167), mean practical hours per semester (13.2), and maximum lecture hours in a single semester (18).
All visualizations rendered without errors within the Streamlit interface and were visually consistent with the platform’s design system.

5. Challenges Faced
The development of this project involved overcoming several practical challenges. Three key difficulties are described below.
5.1 Streamlit State Management
Streamlit reruns the entire Python script from scratch every time a user interacts with the interface. This meant that whenever a student navigated between tabs such as the Roadmap or Analytics page, all their previous selections and any expanded tree branches would reset. To address this, we used Streamlit’s st.session_state feature to persist user selections and UI state across script reruns, so the application feels smooth and continuous rather than resetting on every click. Careful management of session state keys was required to avoid conflicts between the independent states of different application modules.
5.2 ECharts–Python Event Bridging
One of the most technically demanding aspects of the project was capturing JavaScript-side click events from the ECharts tree component and reliably propagating them back to the Streamlit Python backend to update the analytics panel.
The streamlit-echarts library allows a JavaScript function to be registered as an event handler via the events parameter dictionary. When a node is clicked, this function receives the ECharts params object and must return a Python-serializable value. The returned value is then available on the Python side as the return value of the st_echarts() call. The critical challenge was that only JSON-serializable data can cross this JavaScript-to-Python boundary. Since each node in the curriculum tree contains a nested metadata dictionary, we had to ensure the entire node’s data property — including nested keys for Credits, L, T, P, and H — was correctly serialized by the event handler function (return params.data;) and correctly deserialized by Streamlit into a Python dictionary.
A further complication arose from the fact that semester and year group nodes do not carry a metadata field. The application must gracefully distinguish between a click on an expandable group node (which has no analytics to display) and a click on a subject leaf node (which should trigger the analytics panel). This distinction is handled by checking for the presence of the metadata key in the returned dictionary: if selected_node and "metadata" in selected_node.
5.3 Custom CSS Styling in Streamlit
Streamlit’s default appearance is plain and designed for data science tools, not for a polished academic platform. To achieve the editorial look required — with the Newsreader serif font for headings, Work Sans for body text, and a parchment-colored background — custom CSS was injected directly into the app using st.markdown with unsafe_allow_html=True. Getting these styles to apply correctly without breaking Streamlit’s own built-in components required careful inspection of Streamlit’s internal DOM structure and iterative testing. In particular, overriding the background color of metric containers and chart areas required targeting internal Streamlit CSS selectors (e.g., [data-testid="stMetricValue"]) rather than standard HTML element selectors.

Subject Categorization and Color Injection
CATEGORY_COLORS = {
    "Core Subject":     "#5470c6",  # Blue
    "Elective":         "#91cc75",  # Green
    "Project/Research": "#fac858",  # Yellow
    "Lab/Practical":    "#ee6666",  # Red
    "Mathematics":      "#73c0de",  # Light Blue
    "Root/Semester":    "#333333"   # Dark Grey
}

def determine_category(name, metadata):
    name_lower = name.lower()
    if "elective" in name_lower:
        return "Elective"
    elif "project" in name_lower or "research" in name_lower \
            or "internship" in name_lower:
        return "Project/Research"
    elif "lab" in name_lower or "drawing" in name_lower \
            or (metadata.get("L") == 0 and metadata.get("P", 0) > 0):
        return "Lab/Practical"
    elif "mathematics" in name_lower or "math" in name_lower:
        return "Mathematics"
    else:
        return "Core Subject"

def inject_colors(node):
    if "metadata" in node:
        category = determine_category(node["name"], node["metadata"])
        node["category"] = category
        color = CATEGORY_COLORS[category]
        node["itemStyle"] = {"color": color, "borderColor": color}
    else:
        node["category"] = "Root/Semester"
        color = CATEGORY_COLORS["Root/Semester"]
        node["itemStyle"] = {
            "color": "white",
            "borderColor": color,
            "borderWidth": 2
        }
    if "children" in node:
        for child in node["children"]:
            inject_colors(child)  # Recursive traversal

6.3 Analytics Module (analytics.py)
This module structures the curriculum summary data into a Pandas DataFrame, uses NumPy for statistical aggregation, and generates the credit distribution bar chart (Seaborn) and the lecture vs. practical line chart (Matplotlib).

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

data = {
    'Semesters': ['Sem 1','Sem 2','Sem 3','Sem 4',
                  'Sem 5','Sem 6','Sem 7','Sem 8'],

    'Lecture Hours':    [14, 15, 18, 15, 17, 15, 15,  6],
    'Practical Hours':  [10, 12, 10, 10, 10, 12, 18, 24]
}
df = pd.DataFrame(data)

# NumPy aggregations
total_credits      = np.sum(df['Credits'])       # 167
avg_practical      = np.mean(df['Practical Hours'])  # 13.2

fig_bar, ax_bar = plt.subplots(figsize=(8, 5))
sns.barplot(data=df, x='Semesters', y='Credits',
            color='#15236d', ax=ax_bar)

8. Applications
The Scholarly Engineer platform has a wide range of practical applications across educational institutions and academic planning contexts.
University Curriculum Portals: The platform can be deployed as an official departmental resource, replacing static PDF syllabus documents with an interactive, always-current digital curriculum guide with subject-level analytics.
Student Academic Planning: Students can use the roadmap module to understand course progressions, identify workload-intensive semesters, and make informed decisions about elective selections based on the credit and hour analytics.
Faculty Workload Analysis: Educators and administrators can use the analytics dashboard to identify semesters with disproportionately high theoretical or practical loads and adjust curriculum design accordingly.
Multi-Discipline Expansion: The platform’s modular architecture and JSON-driven data model allow it to be replicated for other engineering branches by replacing the underlying curriculum_data.json file without changes to the application framework.
Career Guidance Integration: The Project Hub can be extended to link hardware projects to industry-relevant skills, helping students identify which projects best prepare them for careers in embedded systems, VLSI, and IoT.
EdTech Prototyping: The project demonstrates that Python alone, using Streamlit and its ecosystem, can produce production-quality educational technology applications, including rich interactive charts driven by JavaScript rendering engines, without requiring a separate frontend framework.

9. Future Scope
While the current implementation successfully achieves all objectives set for this Python Programming course project, numerous directions exist for future enhancement.
User Authentication and Progress Tracking: Implementing a login system using Streamlit-Authenticator would allow students to create personal profiles, mark completed courses, and track their individual progress through the curriculum roadmap.
Machine Learning Integration: Integrating scikit-learn could enable predictive models that forecast student workload stress based on historical grading data, course combinations, and semester performance trends.
Dynamic Syllabus Scraping: Automating the Pandas data pipeline to scrape the university website for syllabus updates would remove the need for manually maintained JSON data files and ensure the platform always reflects the current curriculum.
Database Backend: Replacing in-memory Pandas DataFrames with a SQLite or PostgreSQL database backend would enable persistent storage of user progress, annotations, and custom resource links.
Prerequisite Chain Visualization: Extending the tree model to encode and display formal subject prerequisites would help students understand the dependency structure of the curriculum and plan their elective choices more effectively.
Mobile-Responsive Deployment: Deploying the application on Streamlit Cloud with a responsive CSS layout and optimized chart dimensions would make the platform accessible on smartphones and tablets.

10. Conclusion
“The Scholarly Engineer” project was conceived, designed, implemented, and documented successfully as a Python Programming course project by Het Dave and Hemil Thakkar of the Electronics Engineering Department at Birla Vishvakarma Mahavidyalaya, under the guidance of Dipak M Patel.
The project achieved all of its primary objectives. The deliberate architectural decision to consolidate the entire application — data processing, hierarchical tree rendering, and analytics visualization — into a monolithic Python/Streamlit codebase proved to be a sound and maintainable choice. The streamlit-echarts library successfully rendered a fully interactive, color-coded hierarchical curriculum tree; the custom inject_colors() algorithm correctly categorized all curriculum subjects and applied a consistent visual taxonomy; the ECharts–Python event bridge reliably propagated subject click events to trigger subject-level pie chart analytics; the Pandas and NumPy data pipeline structured and aggregated curriculum data accurately; and the Matplotlib and Seaborn visualization modules produced clear, informative charts that transform static syllabus data into actionable academic insights.
The three main challenges encountered — Streamlit’s stateless execution model, the complexity of bridging JavaScript ECharts click events back to the Python backend, and CSS injection for custom theming — were all resolved through systematic problem-solving and iterative development, providing valuable real-world programming experience.
The success of this project demonstrates that Python’s rich ecosystem of libraries enables first-year engineering students to build sophisticated, practically useful, and professionally designed software tools. The project lays a strong foundation for more advanced work in data science, interactive visualization, and full-stack Python development in the subsequent years of the B.Tech programme.
